Contexte :
Dans les architectures modernes, la création d’API REST ou GraphQL constitue un élément
central. Pourtant, une grande partie du travail consiste à répéter des tâches structurelles :
modèles, contrôleurs, validation, documentation, tests, etc. Les générateurs existants sont
puissants mais souvent limités ou complexes à étendre.


Objectif :
Concevoir un outil capable de générer automatiquement un backend complet (modèles,
endpoints CRUD, documentation OpenAPI, tests) à partir d’un simple schéma d’entités défini en
JSON ou YAML.
Travail et résultat attendus :
Il est attendu un travail de recherche de l’existant et un POC illustrant l’intégralité de
l’objectif. Le POC doit être suHisamment fonctionnel pour illustrer sa véritable usabilité en
pratique.
